# OpenClaw Glossary

Quick definitions for common terms used in these docs.

- **Gateway**: the always-on OpenClaw service that handles channels, tools, automation, and the control UI.
- **Workspace**: the folder/files the agent uses as its “home” (notes, memory, code, etc.).
- **Channel**: where messages come from/go to (Telegram, WhatsApp, etc.).
- **Pairing**: approving/connecting a channel or node to your gateway.
- **Token**: a secret used to authenticate to a service/UI.
- **SSH tunnel**: a secure way to access a private port on a server through SSH.
- **Tailscale**: a private networking tool (“VPN-like”) that makes your devices part of the same secure network.
- **UFW**: Ubuntu’s firewall tool.
- **fail2ban**: watches logs and bans IPs that brute-force SSH.
- **unattended-upgrades**: automatically installs security updates.
- **OOM**: “out of memory,” when the server kills a process because RAM is exhausted.
